> **Scope**: Extends J.4 of design-principles.md, adds practice run schedules, timing checkpoints, and emergency strategies. Does NOT repeat the slide count table from J.4.

## Practice Run Schedule

Five runs, each with a different purpose. Don't skip to run 5.

### Run 1 — Rough Pass (1 week before)
**Goal**: Find structural problems before you're attached to the slides.

- Talk through every slide out loud — no skipping
- Don't time yourself yet
- Note: slides that take > 3 min to explain (too dense), slides you skip over (cut them), transitions that feel awkward
- Output: list of slides to cut, merge, or rewrite

### Run 2 — Timed Draft (5 days before)
**Goal**: Get a real time reading. Expect to be 20–30% over.

- Use a stopwatch. Record time per section, not just total.
- Talk as if presenting — full sentences, not notes
- Mark every slide where you stumbled or went blank
- Output: per-section timing, list of problem slides

### Run 3 — Cut and Fix (3–4 days before)
**Goal**: Hit target time. Cut ruthlessly.

- Apply cuts from Run 2: remove slides, compress bullet points, trim examples
- Rule: if a slide doesn't make the talk better, it makes it worse
- Re-time after cuts
- Output: deck at or under target time

### Run 4 — Dress Rehearsal (2 days before)
**Goal**: Simulate real conditions.

- Stand up. Use your clicker. Use the actual display if possible.
- Speak at presentation pace — not reading pace
- Have someone watch (colleague, labmate, anyone)
- Ask them: "What was the main message?" If they can't answer, the talk isn't clear yet
- Output: final list of tweaks, confirmed timing

### Run 5 — Confidence Run (day before or morning of)
**Goal**: Lock in muscle memory. Not for changes.

- Run through once, no stopping
- If something goes wrong, keep going — practice recovering, not perfecting
- No new slides after this run
- Output: confidence, not changes

## Timing Checkpoints — 15-Minute Talk

Use these checkpoints during the actual presentation. Glance at your watch at each transition.

| Checkpoint | Target Time | If Behind | If Ahead |
|------------|-------------|-----------|----------|
| End of introduction | 2:00 | Cut one background slide | Slow down on methods |
| End of background | 4:00 | Merge two results slides | Add one sentence of context per result |
| End of methods | 7:00 | Drop one results figure | Spend more time on implications |
| End of results | 12:00 | Skip to conclusion immediately | Add one "what this means" slide |
| End of conclusion | 15:00 | — | — |

**Buffer rule**: Build 1 min of buffer into your plan. Use it for questions that come mid-talk, not for adding content.

## Timing Checkpoints — 45-Minute Talk

| Checkpoint | Target Time | If Behind | If Ahead |
|------------|-------------|-----------|----------|
| End of introduction | 8:00 | Cut 1–2 background slides | Add real-world example |
| End of background | 15:00 | Compress methods to overview only | Show one extra prior-work comparison |
| End of methods | 25:00 | Drop one results subsection | Add methodological detail |
| Halfway through results | 33:00 | Skip secondary findings | Slow down, add interpretation |
| End of results | 40:00 | Cut discussion to 2 bullets | Full discussion |
| End of conclusion | 45:00 | — | — |

**Checkpoint habit**: Set silent vibration alerts on your phone at each checkpoint time. Glance at your watch, not the clock on the wall — it's harder to read under lights.

## Emergency Strategies

### Running Behind (More Than 2 Minutes Over)

**Don't**: Speed up your speech. The audience loses comprehension and you look panicked.

**Do**:

1. **Cut the next section's detail, not the section itself.** Instead of showing 3 results figures, show 1 and say "I'll skip the other two for time — they're in the paper."
2. **Use the summary shortcut.** Jump to your conclusion slide and say "In the interest of time, let me jump to the key takeaway." Audiences respect this.
3. **Drop backup slides.** If you planned to show sensitivity analysis, don't.
4. **Compress transitions.** Cut "So, as we can see from this figure…" — just state the finding.

**Triage priority** (cut in this order):
- Secondary results
- Methodological detail
- Literature review slides
- Discussion nuance
- **Never cut**: the main result, the conclusion, the call to action

### Running Ahead (More Than 2 Minutes Under)

**Don't**: Add slides you cut during practice. Don't pad with filler.

**Do**:

1. **Slow down on results.** Walk through each figure more carefully. "Let me make sure this is clear — the x-axis here is…"
2. **Add one example.** "To make this concrete, let me give you a specific case…"
3. **Invite early questions.** "I have a few minutes — are there questions before I conclude?"
4. **End on time, not early.** Finishing 3 min early is fine. Finishing 8 min early signals poor preparation.

### Technical Failure (Slides Won't Load / Projector Dies)

**Prepare for this.** It happens more than you think.

1. Have your slides on a USB drive AND in cloud storage (Google Drive, Dropbox)
2. Know your talk well enough to give the first 5 min without slides
3. Keep a PDF backup — it opens on any machine
4. If slides fail completely: "I'll talk through this without slides — the key points are…"

### Audience Running Late (You Start 10 Min Late)

- Don't compress the whole talk — cut the background section
- Skip the outline slide if you have one
- Start with the hook, not the preamble
- Announce: "We're starting late, so I'll focus on the core results"

## Quick Reference Card

| Talk Length | Slides | Practice Runs | Key Checkpoints |
|-------------|--------|---------------|-----------------|
| 5 min (lightning) | 4–5 | 3 minimum | 2:30 (midpoint) |
| 10 min | 8–10 | 3 minimum | 3:00, 7:00 |
| 15 min | 10–12 | 4 minimum | 4:00, 7:00, 12:00 |
| 20 min | 14–16 | 4 minimum | 5:00, 10:00, 16:00 |
| 30 min | 20–24 | 4 minimum | 8:00, 15:00, 24:00 |
| 45 min | 28–35 | 5 minimum | 8:00, 15:00, 25:00, 40:00 |
| 60 min | 35–45 | 5 minimum | 10:00, 20:00, 35:00, 50:00 |

**Pace rule**: 1.5–2 min per slide for technical content. 1 min per slide for lightning talks. Never faster.

**Emergency cut rule**: If running behind, cut 1 slide per 30 seconds over. Don't try to recover time by talking faster.

**Practice-to-talk ratio**: For a 15-min talk, expect 3–4 hours of practice across all runs. For a 45-min talk, expect 8–10 hours. This is not optional — it's what separates good talks from great ones.

## Timing Failure Patterns

Common reasons talks run over — fix these in practice:

| Pattern | Symptom | Fix |
|---------|---------|-----|
| Dense slides | Every slide takes 3+ min | One message per slide |
| Unplanned tangents | "Oh, and another thing…" | Stick to the script for first 3 runs |
| Slow transitions | Long pauses between slides | Practice transitions explicitly |
| Over-explaining figures | "So if you look at the x-axis…" | Title = the finding; figure speaks for itself |
| Late start | Opening remarks, tech setup | Add 2-min buffer to opening |
| Q&A creep | Questions during the talk | "Great question — let's hold for Q&A" |

Common reasons talks run short:

| Pattern | Symptom | Fix |
|---------|---------|-----|
| Skipping context | Audience looks confused | Add one motivating example |
| Rushing results | Key finding unclear | Slow down, state finding twice |
| Cutting practice | Talk feels shorter live | Practice in real conditions |
| Nervous pace | Speaking faster than usual | Breathe. Pause after each slide title. |
